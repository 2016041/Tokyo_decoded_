# 新規案件オーダーシート

> **ディレクターの使い方**
> 1. `_新規案件テンプレート` フォルダを丸ごとコピーし、案件名にリネームする
> 2. このファイルを記入する
> 3. C-1（PM）への最初のメッセージにこのファイルの内容を貼り付ける
> 4. C-1 が Phase 1 を開始し、完了後に承認を求めてくる → 承認で Phase 2 GO
>
> 記入が多いほど壁打ちなしで Phase 1 が完了します。

## C-1（PM）への指示 ※変更不要

上記の情報をもとに、以下を実行してください。

### Step 1 — プロジェクトセットアップ（最初に実施）

以下の `[FILL:]` をオーダーシートの情報で埋める：

| ファイル | 参照する項目 |
|---------|------------|
| `.claude/CLAUDE.md` | §1 案件基本情報（案件名・クライアント・ディレクター・納期・予算） / §3 ページ構成 / §7 技術制約・環境 |
| `.claude/design.md` | §1 案件基本情報（案件名・クライアント・サイト種別） / §2 目的・ターゲット |

### Step 2 — Phase 1 並列実行

以下を **同時に** 各エージェントに指示する：

| エージェント | タスク | 参照する項目 | 出力ファイル |
|------------|--------|------------|------------|
| **C-2**（デザインシステム設計者） | `.claude/design.md` の全 `[FILL:]` を記入し、`app/globals.css` の `@theme inline {}` のカラー・フォントトークンを確定する | §1 案件基本情報 / §2 目的・ターゲット / §4 デザイン方向性（ムード・参考サイト・カラー・ダークモード・フォント・アニメーション） | `.claude/design.md`, `app/globals.css` |
| **C-3**（フロントエンドテクニカルリード） | アーキテクチャ文書を作成する（"use client" 境界・型定義・必要な環境変数リスト） | §3 ページ構成 / §6 機能要件 / §7 技術制約・環境 | `.claude/architecture.md` |
| **C-4**（QA・アクセシビリティ担当） | アクセシビリティ要件定義書を作成する（ARIA・キーボード操作・コントラスト比） | §2 目的・ターゲット / §3 ページ構成 / §6 機能要件 | `.claude/accessibility.md` |
| **C-5**（コンテンツ・SEOストラテジスト） | `content/*.ts` ファイルを `as const` 形式で作成する | §2 目的・ターゲット / §5 コンテンツ / §6 機能要件（OGP・JSON-LD・sitemap） | `content/*.ts` |

### Step 3 — 統合・承認申請

全エージェントの成果物が揃ったら「Codex 向け実装指示書」に統合し、ディレクターに **Phase 2 GO の承認** を求めること。
